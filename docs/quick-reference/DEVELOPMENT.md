# Development Quick Reference

Quick reference guide for common development tasks and commands.

## 🚀 Quick Start

### First Time Setup

```bash
# Clone and setup (one command)
./scripts/dev-setup.sh
```

### Start Development Servers

```bash
# Terminal 1 - Backend
cd backend
source .venv/bin/activate  # Linux/Mac
# OR
source .venv/Scripts/activate  # Windows Git Bash
python manage.py runserver

# Terminal 2 - Frontend
cd frontend
npm run dev
```

### Access Points

| Service | URL | Notes |
|---------|-----|-------|
| Frontend | http://localhost:5173 | React + Vite dev server |
| Backend API | http://localhost:8000 | Django REST API |
| API Docs | http://localhost:8000/swagger/ | Interactive API documentation |
| Django Admin | http://localhost:8000/admin/ | Admin interface |
| Health Check | http://localhost:8000/health/ | System status |

## 📋 Common Tasks

### Backend (Django)

| Task | Command |
|------|---------|
| Run development server | `cd backend && python manage.py runserver` |
| Run tests | `cd backend && pytest` |
| Run specific test | `cd backend && pytest tests/test_file.py::test_name` |

| Apply migrations | `cd backend && python manage.py migrate` |

| Django shell | `cd backend && python manage.py shell` |

| Lint code | `cd backend && flake8 .` |
| Type check | `cd backend && mypy . --ignore-missing-imports` |
| Validate environment | `cd backend && python scripts/validate_env.py` |

| Run unit tests only | `cd frontend && npm run test:unit` |

| Task | Command |

## 🔧 Troubleshooting

### Common Issues

#### Backend Issues

**"Module not found" errors**
```bash
cd backend
source .venv/bin/activate
pip install -r requirements.txt
```

**"No such table" errors**
```bash
cd backend
python manage.py migrate
```

**CORS errors in browser**
- Check `backend/config/settings.py` → `CORS_ALLOWED_ORIGINS`
- Should include `http://localhost:5173` for development

**Database locked (SQLite)**
```bash
cd backend
# Stop all running servers
rm db.sqlite3  # ⚠️ Development only! Deletes data
python manage.py migrate
python create_admin.py
```

#### Frontend Issues

**"Cannot find module" errors**
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
```

**TypeScript errors**
```bash
cd frontend
npx tsc --noEmit  # Check for type errors
```

**Build fails**
```bash
cd frontend
rm -rf dist node_modules
npm install
npm run build
```

#### API Connection Issues

**Frontend can't reach backend**
1. Verify backend is running: http://localhost:8000/health/
2. Check frontend .env: `VITE_API_BASE_URL=http://localhost:8000`
3. Check browser console for CORS errors
4. Verify Django settings allow `localhost:5173`

## 🔐 Environment Variables

### Backend (.env)

```bash
# Copy template and edit
cp backend/.env.example backend/.env

# Edit these required variables:
SECRET_KEY=...           # Auto-generated by dev-setup.sh
ADMIN_PASSWORD=...       # Set a secure password
DEBUG=True               # False in production
DATABASE_URL=...         # PostgreSQL URL (optional, uses SQLite by default)
```

### Frontend (.env)

```bash
# Create frontend/.env
VITE_API_BASE_URL=http://localhost:8000
VITE_ENVIRONMENT=development
```

## 🧪 Testing

### Run All Tests

```bash
# Backend
cd backend && pytest -v

# Frontend
cd frontend && npm test
```

### Run Specific Test File

```bash
# Backend
cd backend && pytest tests/test_courses.py -v

# Frontend
cd frontend && npm test -- CourseService.test.ts
```

### Run Tests with Coverage

```bash
# Backend
cd backend && pytest --cov=core --cov-report=html

# Frontend
cd frontend && npm run test:coverage
```

### Watch Mode (Auto-run on changes)

```bash
# Frontend only
cd frontend && npm test -- --watch
```

## 📦 Database Operations

### Reset Database (Development)

```bash
cd backend

# ⚠️ WARNING: Deletes all data!
rm db.sqlite3
python manage.py migrate
python create_admin.py

# Load sample data (if fixtures exist)
python manage.py loaddata fixtures/sample_data.json
```

### Create Database Backup

```bash
cd backend

# SQLite
cp db.sqlite3 db.backup.sqlite3

# PostgreSQL
pg_dump $DATABASE_URL > backup.sql
```

### Access Database Shell

```bash
cd backend

# SQLite
python manage.py dbshell

# Or use the Django shell
python manage.py shell
>>> from core.models import User, Course
>>> User.objects.all()
```

## 💡 Tips & Best Practices

### Code Quality

- Run `./scripts/pre-deploy-check.sh` before committing
- Format code before committing: `black .` (backend), `npm run format` (frontend)
- Write tests for new features
- Keep commits small and focused

### Performance

- Use Django Debug Toolbar to identify slow queries
- Use React DevTools Profiler to find slow components
- Monitor bundle size: `cd frontend && npm run build` (check dist/ size)

### Security

- Never commit secrets (.env files, API keys, passwords)
- Use environment variables for all configuration
- Validate environment with: `cd backend && python scripts/validate_env.py`
- Rotate credentials regularly in production

### Git Workflow

```bash
# Create feature branch
git checkout -b feature/your-feature-name

# Make changes and commit
git add .
git commit -m "Description of changes"

# Push to remote
git push -u origin feature/your-feature-name

# Create pull request on GitHub
```

### Backend
